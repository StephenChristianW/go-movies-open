Go + Gin + Swag 使用记录

1. 安装工具

# 安装 swag CLI
go install github.com/swaggo/swag/cmd/swag@v1.6.0

# 安装 Gin Swagger 中间件和 Swagger UI
go get github.com/swaggo/gin-swagger@v1.6.0
go get github.com/swaggo/files@v0.9.0

注意：新版本1.7.0需要 Go 1.24+，低版本用 v1.6.0

2. 初始化 Swagger 文档

# 在项目根目录执行
swag init -g StacBuilder.go
  • -g 指定 main 或入口 handler 文件
  • 生成 docs/ 文件夹，包含 swagger.json、swagger.yaml 和 docs.go

3. 配置 Gin 路由

import (
    _ "项目路径/docs"           // 引入自动生成的 docs
    swaggerFiles "github.com/swaggo/files"
    ginSwagger "github.com/swaggo/gin-swagger"
)

router := gin.Default()
router.GET("/swagger/*any", ginSwagger.WrapHandler(swaggerFiles.Handler))

访问：http://localhost:8080/swagger/index.html

4. Controller 注释规范

// @Summary 生成单个邀请码
// @Description 生成单个邀请码
// @Tags InviteCode
// @Accept json
// @Produce json
// @Success 200 {object} controller.SuccessResponseDoc
// @Failure 422 {object} controller.ErrorResponseDoc
// @Router /invite/create_one [post]
func (InviteCodeHandler) GenerateAndInsertCode(c *gin.Context) {}

5. 更新文档

swag init -g StacBuilder.go

每次新增或修改接口，重新执行即可更新 docs

6. 响应类型

type SuccessResponseDoc struct {
    Code int         `json:"code"`
    Msg  string      `json:"msg"`
    Data interface{} `json:"loginData"`
}

type ErrorResponseDoc struct {
    Code int    `json:"code"`
    Msg  string `json:"msg"`
}

在注释里引用：@Success 200 {object} controller.SuccessResponseDoc

7. 注意事项

  • swag 对类型定义严格，必须能在编译期找到类型
  • body 参数必须用 struct，不能直接写 map 或 JSON
  • Gin-swagger 对 Go 版本敏感，低于 1.24 的 Go 需要旧版本
